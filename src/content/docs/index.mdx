## なぜ Hapbeat？

<CardGrid>
  <Card title="低遅延・直結通信" icon="rocket">
    SDK から Wi-Fi UDP broadcast で **Hapbeat に直接送信**。Bridge やクラウドを経由しないため、ゲーム / VR で重要な数 ms オーダーのレスポンスを実現。
  </Card>
  <Card title="クラウド非依存・オフライン動作" icon="seti:lock">
    App ID / API キー不要、ネット接続不要で動作。展示ブースやライブ会場、VR ヘッドセット単体でも安定して使えます。
  </Card>
  <Card title="運用しやすい Kit 構造" icon="document">
    触覚コンテンツ（波形・タイミング）はアプリやファームから分離された **Kit** として配布。Kit を差し替えるだけでコンテンツ更新でき、ファーム書換やアプリ再ビルド不要。
  </Card>
  <Card title="マルチプレイヤー対応" icon="puzzle">
    **Group ID** によるルーティングで、同じ空間に複数台の Hapbeat があっても混信なし。展示の同時体験やマルチプレイヤー VR にそのまま使えます。
  </Card>
  <Card title="オープンな仕様と SDK" icon="open-book">
    Wi-Fi UDP プロトコルや Kit フォーマットは [Contracts](/docs/contracts/overview/) で公開済み。Unity / Unreal / TouchDesigner / Max 等への拡張も自前で実装可能です。
  </Card>
  <Card title="ESP32-S3 固定ランタイム" icon="setting">
    本体ファームウェアはユーザー操作なしで動く固定ランタイム。コンテンツは Kit、ロジックはアプリ／SDK 側に分離しているため、本体が壊れにくい設計です。
  </Card>
</CardGrid>

## ユースケース

Hapbeat はゲーム、VR、ライブ・展示など **「身体に直接触覚を返したい」** あらゆる場面で使えます。

<CardGrid>
  <Card title="VR / AR / MR ゲーム" icon="laptop">
    Quest / PC VR / モバイル VR から直接振動を発火。Unity SDK の Trigger コンポーネントで衝突 / 入力 / アニメーションに紐付けるだけ。
  </Card>
  <Card title="インスタレーション・体験型展示" icon="seti:image">
    オフライン動作 + マルチデバイス対応で、ブース／会場規模の常設展示にも投入可能。Kit 差し替えで演目を即時切替。
  </Card>
  <Card title="音楽 / リズム体験 / ライブ演出" icon="seti:audio">
    オーディオに同期した振動を SDK 経由でリアルタイム送信。ライブ会場での観客同期演出やリズムゲームに。
  </Card>
  <Card title="ヘルスケア・トレーニング" icon="heart">
    呼吸・姿勢・心拍などのフィードバック触覚として活用可能。低遅延・無線・装着型なので運動中の使用にも適します。
  </Card>
</CardGrid>

> 製品の詳細・実例は本家 [hapbeat.com](https://hapbeat.com) を参照してください。
