## 特徴

<CardGrid>
  <Card title="直結通信" icon="rocket">
    SDK は Wi-Fi UDP broadcast でデバイスへ直接送信する。Bridge やクラウドを経由しないため遅延が短い。
  </Card>
  <Card title="オフライン動作" icon="seti:lock">
    App ID / API キー不要、ネット接続なしで動作する。展示や VR ヘッドセット単体運用にも対応。
  </Card>
  <Card title="Kit によるコンテンツ分離" icon="document">
    触覚コンテンツ（波形・タイミング）は Kit として配布する。アプリやファームの再ビルドを行わずに更新できる。
  </Card>
  <Card title="複数台ルーティング" icon="puzzle">
    Group ID 指定により、同一空間に複数の Hapbeat があっても混信なくルーティングできる。
  </Card>
  <Card title="公開仕様" icon="open-book">
    Wi-Fi UDP プロトコルと Kit フォーマットは [Contracts](/docs/contracts/overview/) に文書化されている。独自 SDK / 拡張も実装可能。
  </Card>
  <Card title="固定ランタイム" icon="setting">
    デバイス側ファームウェアはユーザーカスタマイズを前提としない。コンテンツは Kit、ロジックは SDK 側に分離されている。
  </Card>
</CardGrid>

## ユースケース

Hapbeat は身体に直接触覚フィードバックを返す用途全般で利用できます。

<CardGrid>
  <Card title="VR / AR / MR" icon="laptop">
    Quest / PC VR / モバイル VR から発火。Unity SDK の Trigger コンポーネントで衝突・入力・アニメーションに紐付けられる。
  </Card>
  <Card title="インスタレーション・展示" icon="seti:image">
    オフライン動作 + 複数台ルーティングにより、ブース／会場規模の常設展示に対応。Kit 差し替えで演目を切替。
  </Card>
  <Card title="音楽 / リズム / ライブ" icon="seti:audio">
    オーディオに同期した振動を SDK 経由でリアルタイム送信。ライブ会場の同期演出やリズムゲームで利用される。
  </Card>
  <Card title="ヘルスケア / トレーニング" icon="heart">
    呼吸・姿勢・心拍などのフィードバック触覚として利用可能。ワイヤレスかつ装着型のため運動中の使用にも適する。
  </Card>
</CardGrid>
