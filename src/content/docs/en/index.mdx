## Why Hapbeat?

<CardGrid>
  <Card title="Low-latency, direct comms" icon="rocket">
    The SDK sends events to Hapbeat via Wi-Fi UDP broadcast — **no bridge, no cloud** — for the millisecond-class response that games and VR demand.
  </Card>
  <Card title="Cloud-free, offline-ready" icon="seti:lock">
    No App ID, no API key, no internet required. Works reliably at exhibition booths, live venues, or on a standalone VR headset.
  </Card>
  <Card title="Operate via Kits" icon="document">
    Haptic content (waveforms, timings) ships as a **Kit**, decoupled from app and firmware. Swap a Kit to update content — no firmware reflash, no app rebuild.
  </Card>
  <Card title="Multi-player ready" icon="puzzle">
    **Group ID** routing keeps multiple Hapbeats in the same space from interfering. Drop-in support for shared exhibits and multiplayer VR.
  </Card>
  <Card title="Open spec & SDKs" icon="open-book">
    The Wi-Fi UDP protocol and Kit format are documented in [Contracts](/en/docs/contracts/overview/). Build your own Unity / Unreal / TouchDesigner / Max integrations.
  </Card>
  <Card title="Stable fixed runtime" icon="setting">
    The on-device firmware is a fixed runtime that needs no user maintenance. Content lives in Kits, logic in your app/SDK — robust against everyday use.
  </Card>
</CardGrid>

## Use cases

Hapbeat fits anywhere you want to **deliver tactile feedback to the body** — games, VR, live, exhibitions, and beyond.

<CardGrid>
  <Card title="VR / AR / MR games" icon="laptop">
    Fire haptics directly from Quest / PC VR / mobile VR. The Unity SDK's Trigger components attach to collisions, inputs, or animation events with no boilerplate.
  </Card>
  <Card title="Installations & experiential exhibits" icon="seti:image">
    Offline operation and multi-device support make Hapbeat usable for booth-scale or venue-scale permanent installations. Swap a Kit to change the show.
  </Card>
  <Card title="Music / rhythm / live performance" icon="seti:audio">
    Send vibration in sync with audio in real time via the SDK. Audience-sync effects at concerts, rhythm games, and immersive listening experiences.
  </Card>
  <Card title="Health & training" icon="heart">
    Use haptics for breath / posture / heart-rate feedback. Wireless and wearable, suitable for use during physical activity.
  </Card>
</CardGrid>

> See [hapbeat.com](https://hapbeat.com) for product details and case studies.
