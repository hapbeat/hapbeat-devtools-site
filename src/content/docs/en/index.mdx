## Characteristics

<CardGrid>
  <Card title="Direct communication" icon="rocket">
    The SDK sends events to devices over Wi-Fi UDP broadcast — no bridge, no cloud — keeping latency short.
  </Card>
  <Card title="Offline operation" icon="seti:lock">
    No App ID, no API key, no internet required. Suitable for exhibits and standalone VR headset use.
  </Card>
  <Card title="Content separation via Kit" icon="document">
    Haptic content (waveforms, timings) ships as a **Kit**. Updates can be applied without rebuilding the app or reflashing firmware.
  </Card>
  <Card title="Multi-device routing" icon="puzzle">
    Group ID routing prevents interference between multiple Hapbeats in the same space.
  </Card>
  <Card title="Open spec" icon="open-book">
    The Wi-Fi UDP protocol and Kit format are documented in [Contracts](/en/docs/contracts/overview/). You can implement your own SDK or extensions.
  </Card>
  <Card title="Fixed runtime" icon="setting">
    The device firmware is a fixed runtime — content lives in Kits, logic in your app or SDK. The runtime itself is not customized per project.
  </Card>
</CardGrid>

## Use cases

Hapbeat fits any context where you want to deliver tactile feedback to the body.

<CardGrid>
  <Card title="VR / AR / MR" icon="laptop">
    Fire haptics from Quest / PC VR / mobile VR. The Unity SDK's Trigger components attach to collisions, inputs, or animation events.
  </Card>
  <Card title="Installations & exhibits" icon="seti:image">
    Offline operation and multi-device routing support booth- or venue-scale installations. Swap a Kit to change the show.
  </Card>
  <Card title="Music / rhythm / live" icon="seti:audio">
    Send vibration in sync with audio in real time via the SDK. Audience-sync effects at concerts, rhythm games, and immersive listening.
  </Card>
  <Card title="Health & training" icon="heart">
    Use haptics for breath / posture / heart-rate feedback. Wireless and wearable for use during physical activity.
  </Card>
</CardGrid>
