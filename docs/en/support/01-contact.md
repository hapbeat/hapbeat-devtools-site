Hapbeat support is handled through GitHub. Choose the appropriate channel based on your needs.

## Questions, Discussions & Ideas — GitHub Discussions

For "how do I use this?", feature requests, sharing your work, or general conversation, use Discussions.

- **Discussions**: <https://github.com/orgs/hapbeat/discussions>

Categories:

- **Q&A** — Usage questions. Answers can be marked as accepted, making them easy to find later.
- **Ideas** — Feature requests and improvement suggestions.
- **Show and tell** — Share projects or demos made with Hapbeat.
- **General** — Anything else.

You can post with any existing GitHub account — no additional registration required.

## Bug Reports — Repository Issues

For clear malfunctions such as incorrect behavior or crashes caused by code, open an issue in the relevant repository. Include reproduction steps and environment details to help us respond faster.

| Target | Repository | Issues |
|---|---|---|
| Web design tool (Studio) | hapbeat-studio | <https://github.com/hapbeat/hapbeat-studio/issues> |
| Unity SDK | hapbeat-unity-sdk | <https://github.com/hapbeat/hapbeat-unity-sdk/issues> |
| Helper (daemon) | hapbeat-helper | <https://github.com/hapbeat/hapbeat-helper/issues> |
| Specification (Contracts) | hapbeat-contracts | <https://github.com/hapbeat/hapbeat-contracts/issues> |
| This site | hapbeat-devtools-site | <https://github.com/hapbeat/hapbeat-devtools-site/issues> |

If you are unsure which repository to use, or your question involves components not listed above (device firmware, Kit Tools, etc.), post in Discussions and we will route it to the right issue or respond directly.

## Security Issues

Please **do not report security vulnerabilities in public Issues**. Use GitHub's private vulnerability reporting feature instead (Security tab of the relevant repository).
