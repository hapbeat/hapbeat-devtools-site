## 接続関連

<Faq question="Studio でデバイスが見つからない">

- ファームウェアが書き込まれ、Wi-Fi 設定が済んでいるか確認
- PC とデバイスが同じネットワークに繋がっているか
- ファイアウォールで UDP broadcast と mDNS がブロックされていないか
- Helper が起動しているか（Studio 上部の「Helper 接続中」インジケータ）
- Helper のログを確認: `hapbeat-helper log`（[](/docs/tools/helper/getting-started/)）

</Faq>

<Faq question="Wi-Fi 接続できない">

- Hapbeat は 2.4 GHz 帯のみ対応。5 GHz 専用ルーターでは動作しない
- Studio の Manage タブから Wi-Fi を再設定: [](/docs/tools/studio/initial-setup/)

</Faq>

<Faq question="VR HMD（Quest 等）から直接使いたい">

- Wi-Fi UDP broadcast 方式なので HMD が同じネットワークにあれば動く
- HMD が AP 機能を持たないためルーターなしでは Hapbeat を AP にする必要あり

</Faq>

<Faq question="PC に Wi-Fi と有線 LAN を同時接続するとデバイスがオフラインになる">

別途センサーなどを有線 LAN（イーサネット）で繋ぎ、Hapbeat を Wi-Fi で使う構成で起こります。

**原因**: Hapbeat の通信は UDP broadcast を使います。PC を Wi-Fi と有線 LAN の **2 つの別ネットワーク**に同時接続すると、Windows は通常**有線を優先**するため、broadcast が有線側（センサーのネットワーク）にのみ送出され、Wi-Fi 上の Hapbeat に届きません。Studio がオフライン表示になるだけでなく、同じ PC からの触覚再生にも影響します。

**対処**: Wi-Fi アダプターの**インターフェイスメトリック**を有線より小さく（優先度を高く）します。センサーへの通信は宛先指定（unicast）のため、変更後も問題なく動作します。

管理者権限の PowerShell:

```powershell
# 変更前の値を控える
Get-NetIPInterface -AddressFamily IPv4 | ft InterfaceAlias,InterfaceMetric

# Wi-Fi を優先に（エイリアス名は環境に合わせる）
Set-NetIPInterface -InterfaceAlias "Wi-Fi" -InterfaceMetric 10
Set-NetIPInterface -InterfaceAlias "イーサネット" -InterfaceMetric 60
```

元に戻す場合:

```powershell
Set-NetIPInterface -InterfaceAlias "Wi-Fi" -AutomaticMetric Enabled
Set-NetIPInterface -InterfaceAlias "イーサネット" -AutomaticMetric Enabled
```

GUI の場合: アダプター設定 → Wi-Fi の IPv4 プロパティ → 詳細設定 → 「自動メトリック」のチェックを外し、Wi-Fi に小さい値を設定（戻すときはチェックを戻す）。

> 有線（センサー）と Wi-Fi（Hapbeat）は別サブネットになるようご設定ください。将来的には Studio / Helper / SDK 側で送信インターフェイスを選べるよう改善予定です。

</Faq>

## Kit / コンテンツ関連

<Faq question="Studio でビルドした Kit が転送できない">

- Helper が起動中か、WebSocket（`localhost:7703`）に接続できているか
- Studio 上部の「Helper 接続中」インジケータを確認
- 対象デバイスがオンラインで Studio から見えているか確認

</Faq>

<Faq question="振動が鳴らない">

- Kit が転送されているか（Studio の Kit タブで対象デバイスへの転送状態を確認）
- Event ID が合っているか
- デバイスの音量（Volume）が十分高いか

</Faq>

## その他

<Faq question="複数台同時に別々の振動を出したい">

**Group ID** 機能で分離する。[](/docs/concepts/group-player-addressing/) 参照。

</Faq>
